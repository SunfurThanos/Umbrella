Umbrella X2

Nuevo FrameWork web

Umbrella es un FrameWork inspirado en Adobe Flash Player cuya finalidad es otorgar una gran productividad a la hora de desarrollar paginas web, brindando una gran experiencia a los programadores & usuarios web, para de esa manera generar un equilibrio de bienestar genérico, aunque este Framework esta en desarrollo demuestra su enorme potencial a corto plazo.



Herramientas en desarrollo


PERFORMANCE: mejora del 32% para el Framework a la hora indexar & refactorizar celdas, dando una mayor diferencia en la experiencia positiva al usuario web final


Progreso para imágenes:

  • soporte para la propiedad background-image de css.
  • Botón de cancelar & reanudar carga.
  • Detectar cuando el usuario cambia la url de la celda tipo imagen desde Js.
  • Detectar cuando el usuario ha insertado después de la carga inicial, una nueva celda tipo imagen.

mensajes de leyendas con DECENCIA: con Umbrella podrás crear tooltips, es decir seguirás usando la etiqueta default de HTML llamada title, con la diferencia que si modificas desde CSS el objeto title, Umbrella lo tomara como una clase css privada del propio FrameWork y otorgara por primera vez una forma !intuitiva & fácil! de modificar el estilo de la mencionada Herramienta...

Las buenas noticias aun no terminan, la idea es hacer que los tooltips se puedan implementar en cualquier web, seamos francos a muchos les molesta los mensajes & otros los odian a muerte, !ESO LLEGARA A SU FIN!, para que el usuario pueda verlos, tendrá que presionar la tecla ctrl & sin soltar la tecla posicionar el cursor sobre todos los objetos donde desee ver las leyendas, o bien posicionar el cursor sobre el objeto deseado & presionar la tecla mencionada, al soltar la tecla o mover el cursor fuera del objeto el mensaje se oculta, para dispositivos móviles, las leyendas se podrán ver solo si el usuario, hace el gesto del slider horizontal hacia la izquierda sobre el objeto, al hacer el gesto mencionado pero hacia la derecha o mover el dedo fuera de la leyenda u objeto, !PUFF desaparece el mensaje!, ¿que opinas, excelente o nada que ver, te gusta & quieres contribuir con una idea?, la idea con este Framework es mejorar la tecnología web pero no de forma tiránica o hostil :)


ImageProgress-autoSize: ahora Umbrella podrá detectar de forma automática el tamaño de la imagen antes de que se cargue, para así calcular de forma automática el tamaño que tendrá la celda de progreso de la imagen deseada, antes de generar la pre-carga de cada imagen, guarda todas la url de las celdas de imágenes capturadas en un Array y las enviá al servidor para que averigüe la resolución de cada imagen, el cliente espera que termine, una vez que el servidor termine con su trabajo, el cliente construye cada celda de progreso pero insertándole la resolución de la determinada imagen en pre-cache, manteniendo las propiedades de auto-proporción que caracteriza a las celdas tipo imagen, esta implementación solo sera compatible con servidores con PHP, ¿que opinas, te gusta, te conviene este método?


Imágenes Gif silenciosas: Chrome y las versiones más recientes de Firefox, detienen el ciclo de reproducción de las imágenes Gif, cuando estos no son visibles al deslizar el scroll, aunque hacen su trabajo !MUY bien!, se puede hacer mejor, !MUCHO MEJOR!...

Para empezar Umbrella detendrá de forma automática, el ciclo de reproducción, cuando más del 40% de la imagen no sea visible, cuando sea de nuevo visible en su totalidad se reanudara la reproducción, este método es mucho más efectivo para cuando te deslizas con el scroll donde hay muchos Gif cargados, si haces foco en otro programa, todas las animaciones se detendrán, en la propia celda de la imagen habrán botones para detener, reanudar, descargar & moverte entre fotogramas, si la animación esta en reproducción & haces clic en otro lugar de la pagina la animación se detendrá, para reanudarla solo debes posar una vez el cursor sobre la celda de la animación, ¿que opinas, bueno o arbitrario?


Configuración de imágenes: ventana de configuración inspirada en Adobe flash Player video Settings, que esta dirigido al usuario final & así !otorgarle poder decisión (democracia), cosa que parece que a nadie le importa lo que les pasa a ciertos usuarios, que no disponen de un buen monitor o conexión a Internet fiable!, en dicha ventana podrás hacer cambios aplicados a la imagen concurrente o a todas las imágenes ya sea en un formato especifico o todos los formatos seleccionados, las configuraciones a realizar podrán ser : brillo, saturación, contraste, negar la carga & renderizado bloqueo total, negar la carga automática & dejar que el usuario decida si desea cargar & rasterizar la imagen, bloquear auto reproducción de imágenes Gif & sea el usuario quien decida reproducirlos, conversión y descargar de vectores SVG seleccionados a png, descarga efectiva de todas imágenes de la pagina en forma recursiva o normal, Firefox es el único que trae algo parecido pero siendo honesto hace su trabajo en algunas ocasiones bastante mal aun cuando la implementación es nativa !lo cual deja mucho que pensar!, lo mágico de todo lo mencionado es que dichas configuraciones el navegador las guardara durante 30 días o hasta el infinito, es decir la interacción entre paginas con esta tecnología hará que crezca de forma exponencial el vencimiento de la configuración, quiero que Umbrella mantenga a todos contentos, ¿a vosotros usuarios y colaboradores de Umbrella, están de acuerdo con esto o creen que incluso faltan más cosas, o sencillamente los perjudica?, ¿creen que hay otra manera de equilibrar la balanza entre bienestar, lujo & seguridad?...

Para mostrar el menú flotante donde se puede acceder a esta opción el usuario, tendrá que presionar la tecla ctrl + clic derecho, para hacer desaparecer el menú mencionado solo deberás presionar cualquier tecla o hacer clic en el botón de la "X".


XSprites-CSS: esta implementación QUITA del trono a la técnica actual de los sprites-css, básicamente solo deberás importar desde css de forma independiente los archivos de iconos deseados & Umbrella hará magia para hacer que cada icono este en el cache del navegador pero ademas solo hará una sola petición al servidor para leer un solo archivo, ya con esta acción todos los iconos involucrados serán cargados & rasterizados de forma auto-mágica donde corresponda como si se tratara de un solo archivo. !no es auto-magia, es simple lógica!...

Es muy simple de entender, al hacer la petición al servidor en realidad solo le manda un Array con la dirección de todos los iconos, el servidor lee los archivos y los guarda en una BD virtual muy ligera & limpia en la memoria RAM, este sistema BD a desarrollar la llamare InciteData, luego esta se la manda al cliente, después de enviarla el servidor destruye de la RAM todo rastro de la BD, el cliente leerá de la BD la estructura binaria de cada imagen y la pintara en donde corresponda, por supuesto en el proceso donde el cliente esta recibiendo los datos este mostrara una barra de progreso que se vera de forma duplicada en cada celda involucrada, al finalizar destruye de la RAM la BD virtual por el lado del cliente, ¿tienes algo que preguntar o agregar?.


Menú automático de contenido: la generación de menús interactivos de contenido puede a veces ser tedioso, !LLEGO A SU FIN!, lo único es que tendrás que declarar los titulo de párrafos, post o información vital con la etiqueta h5, el Framework te generara un potente menú de interacción de contenido...

Para mostrar el menú flotante donde se puede acceder a esta opción el usuario, tendrá que presionar la tecla ctrl + clic derecho, una vez hecho esto, para este caso seleccionamos la opción de Menú de contenido, se abrirá una ventana flotante en la derecha con todas las direcciones de contenido, por-supuesto desde CSS vaya que podrás darle estilo al menú mencionado, ¿que opinas, algo que agregar?



TextResponsive: ahora el texto en las paginas web tendrá un tamaño automático, acorde a las circunstancias del tamaño del navegador, es decir un sistema responsive automático.

Style DualCore: permite de forma automática acelerar & cargar en segundo plano de forma paralelas múltiples archivos de estilos, permitiendo acelerar el tiempo de carga de la pagina, sin sacrificar el sistema de cache.

Responsive X: permite un sistema futurista de reglas responsive auto-adaptativo, de las cuales no quiero hablar mucho, ya que quiero que sea !una sorpresa genial e impactante!.

Diseño CSS: múltiples clases css para diseño: botones, barra de cargas, sliders, windows-popup, ListBox, ListIcons, Menubars, ItemsMenus, extensa galería de canvas, efectos de animaciones & calendarios interactivos


Auto-manifiesto de cache: generación y ejecución automática del manifiesto de cache a partir de la interacción del usuario, solo que este sera un archivo virtual basado en cookies permanentes, que se basan en los conceptos de IA para video juegos, esto permitiera una mayor experiencia de usuario para las compañías bancarias entre otros beneficiarios web...

La idea es que esta herramienta de guardar cache o no sea una herramienta gráfica para el usuario final, ¿no crees que el usuario deba tener ese poder, crees que es correcto pero ademas quieres que sea automático, o le pregunte al usuario si desea guardar los datos?, para mostrar el menú flotante donde se puede acceder a esta opción el usuario, tendrá que presionar la tecla ctrl + clic derecho.
